我的配置

(require 'ox-publish);;;注意我这里用是ox-publish而不是org-publish，和Emacs以及Org版本有关
(setq org-publish-project-alist
      '(
        ("blog-notes"
         :base-directory "~/.org/blog/"
         :base-extension "org"
         :publishing-directory "~/.org/superbear.github.io/"
         :recursive t
         :publishing-function org-html-publish-to-html
         :headline-levels 4
         :section-numbers nil
         :auto-preamble t
         :auto-sitemap t                ; Generate sitemap.org automagically...
         :sitemap-filename "sitemap.org"  ; ... call it sitemap.org (it's the default)...
         :sitemap-title "Sitemap"         ; ... with title 'Sitemap'.
         :sitemap-file-entry-format "[%t] %d"
         :author "superbear"
         :html-doctype "html5"
         :html-head "<link rel=\"stylesheet\" href=\"css/worg.css\" type=\"text/css\"/>";;;官网css
         :html-preamble nil
         )
        ("blog-static"
         :base-directory "~/.org/blog/"
         :base-extension "css\\|js\\|png\\|jpg\\|gif\\|pdf\\|mp3\\|ogg\\|swf"
         :publishing-directory "~/.org/superbear.github.io/"
         :recursive t
         :publishing-function org-publish-attachment)
         ("blog" :components ("blog-notes" "blog-static"))
        ;;
        ))

遇到的问题：

  1. 上述导出设置中，在根目录下建多级文件夹，导出后发现css路径不对，怎么办？

    原因 ：引入的css文件路径是相对路径（相对于当前导出文件）

    1). 法一：可以在当前导出文件中配置HTML_HEAD选项（优先级较全局配置高）CSS-support 

    #+HTML_HEAD: <link rel="stylesheet" type="text/css" href="/path/to/worg.css"/>
;;;直接用线上css文件，如官网的这个，就不存在这个问题了，但这个不支持https，需要稍作改动
;;;#+HTML_HEAD: <link rel="stylesheet" type="text/css" href="http://orgmode.org/worg/style/worg.css"/>

    2). 法二：通过选项INCLUDE引入外部文件， 如公用头部，粒度可以细到行，可以在里面加配置项，如 HTML_HEAD、style Include-files 

    #+INCLUDE: "/path/to/common_header.org"

  2. 外部链接默认是当前页面打开，想加上 "target=_blank" 等属性怎么加？

    1).属性加在链接的上一行。如链接前有 list 样式，这句就不生效了。。。Links in HTML export 

    #+ATTR_HTML: :title Github :target _blank
[[https://www.github.com][Github]]

    2). 多次用到某一链接，还可以使用缩写功能，能替换相应内容，甚是方便。Link-abbreviations 

    #+LINK: Org-mode  http://orgmode.org/manual/%s
[[Org-mode:Hyperlinks.html#Hyperlinks][Hyperlinks]]
=>:[[http://orgmode.org/manual/Hyperlinks.html#Hyperlinks][Hyperlinks]]
